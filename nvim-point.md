# nvim 学習ポイント集

---

## Q1. nvimの補完候補を操作するキーバインドは？

**問題:** 補完メニューが表示されたとき、第一候補を選ぶ・他の候補を選ぶ・キャンセルするにはどうする？

**回答:**

| キー | 動作 |
|------|------|
| `<C-y>` | 現在選択中の候補を確定 |
| `<Tab>` | 第一候補を確定（プラグイン依存） |
| `<C-n>` | 次の候補へ（↓） |
| `<C-p>` | 前の候補へ（↑） |
| `<C-e>` | 補完をキャンセルして閉じる |

**ワンポイントアドバイス:**
`<C-e>` は「escape from completion」と覚えると忘れにくい。補完を無視したいときは `<Esc>` より `<C-e>` の方が、インサートモードを維持したまま閉じられるので実用的。

## Q2. LSP・診断（Diagnostics）系のコマンド

**問題:** エラーや警告を確認・移動するコマンドは？

**回答:**

### 診断系
| コマンド | 動作 |
|---|---|
| `Space c d` | 現在行のエラー詳細表示 |
| `[d` | 前のエラーへ移動 |
| `]d` | 次のエラーへ移動 |
| `Space x x` | 診断一覧（Trouble） |
| `Space c q` | quickfix リストを開く |

### LSP系
| コマンド | 動作 |
|---|---|
| `K` | ホバー（型情報・ドキュメント表示） |
| `g d` | 定義へジャンプ |
| `g r` | 参照一覧 |
| `Space c a` | コードアクション（自動修正候補） |
| `Space c r` | リネーム |

**ワンポイントアドバイス:**
`Space c a`（コードアクション）でLSPの自動修正候補を表示できる。キーバインドはプラグイン構成（LazyVimなど）によって異なる場合がある。

## Q3. lazy.nvim の「Could not resolve host」エラーとは？

**問題:** Neovim 起動時に以下のエラーが表示された。何が原因でどう対処する？
```
curl: (6) Could not resolve host: api.github.com
```

**回答:**

| 項目 | 内容 |
|------|------|
| エラー種別 | ネットワーク・DNS解決失敗（curl exit code 6） |
| 原因 | ネット未接続時に lazy.nvim が自動でプラグイン更新チェックを試みる |
| 影響 | エラー表示のみ。nvim本体の動作に支障なし |

**対処法:**
1. ネット接続を確認する（`ping google.com`）
2. 自動チェックを無効化する（設定変更）：
```lua
require("lazy").setup({
  checker = {
    enabled = false,  -- 自動更新チェックを無効化
  },
})
```

**ワンポイントアドバイス:**
`curl: (6)` は「DNSでホスト名を解決できなかった」エラーコード。ネットが繋がっていれば自然に消えるので、オフライン環境で使うことが多い場合は `checker.enabled = false` にしておくと快適。

## Q4. コード編集・補完を効率化するプラグインは？

**問題:** neovimでコード編集・補完を効率化するために導入すべきプラグインは何か？

**回答:**

### 補完系
| プラグイン | 役割 |
|---|---|
| `nvim-cmp` | 補完エンジン（デファクトスタンダード） |
| `LuaSnip` | スニペット展開 |
| `cmp-nvim-lsp` | LSPと補完を繋ぐブリッジ |

### LSP系
| プラグイン | 役割 |
|---|---|
| `nvim-lspconfig` | LSPの設定簡略化 |
| `mason.nvim` | LSPサーバーのインストール管理 |
| `mason-lspconfig.nvim` | MasonとLSPを自動連携 |

### 構文解析系
| プラグイン | 役割 |
|---|---|
| `nvim-treesitter` | 高精度なシンタックスハイライト・インデント |
| `nvim-treesitter-textobjects` | 関数・クラス単位の選択・移動 |

### 導入優先順位
```
1. mason.nvim → LSPサーバーを簡単にインストール
2. nvim-lspconfig → 定義ジャンプ・エラー表示
3. nvim-cmp + LuaSnip → コード補完
4. nvim-treesitter → 正確なハイライト
```

**ワンポイントアドバイス:**
LazyVimを使っている場合はこれらが最初から組み込まれている。一から構築する場合は `mason.nvim` から始めると、LSPサーバーをGUI感覚でインストールできるので導入ハードルが低い。
