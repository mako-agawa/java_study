# nvim 学習ポイント集

---

## Q1. nvimの補完候補を操作するキーバインドは？

**問題:** 補完メニューが表示されたとき、第一候補を選ぶ・他の候補を選ぶ・キャンセルするにはどうする？

**回答:**

| キー | 動作 |
|------|------|
| `<C-y>` | 現在選択中の候補を確定 |
| `<Tab>` | 第一候補を確定（プラグイン依存） |
| `<C-n>` | 次の候補へ（↓） |
| `<C-p>` | 前の候補へ（↑） |
| `<C-e>` | 補完をキャンセルして閉じる |

**ワンポイントアドバイス:**
`<C-e>` は「escape from completion」と覚えると忘れにくい。補完を無視したいときは `<Esc>` より `<C-e>` の方が、インサートモードを維持したまま閉じられるので実用的。

## Q2. LSP・診断（Diagnostics）系のコマンド

**問題:** エラーや警告を確認・移動するコマンドは？

**回答:**

### 診断系
| コマンド | 動作 |
|---|---|
| `Space c d` | 現在行のエラー詳細表示 |
| `[d` | 前のエラーへ移動 |
| `]d` | 次のエラーへ移動 |
| `Space x x` | 診断一覧（Trouble） |
| `Space c q` | quickfix リストを開く |

### LSP系
| コマンド | 動作 |
|---|---|
| `K` | ホバー（型情報・ドキュメント表示） |
| `g d` | 定義へジャンプ |
| `g r` | 参照一覧 |
| `Space c a` | コードアクション（自動修正候補） |
| `Space c r` | リネーム |

**ワンポイントアドバイス:**
`Space c a`（コードアクション）でLSPの自動修正候補を表示できる。キーバインドはプラグイン構成（LazyVimなど）によって異なる場合がある。
